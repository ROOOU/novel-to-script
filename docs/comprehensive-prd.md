# NovelScript SaaS 平台综合 PRD

> 文档类型：产品需求文档（PRD）  
> 更新日期：2026-03-24  
> 版本：v3.0  
> 一句话定位：面向个人创作者的 AI 剧作生产工作台 — 用户从小说或剧本出发，在同一项目中逐步生成分析、大纲、剧本与分镜，并保留浏览、下载、版本迭代与未来视频生成能力。

---

## 1. 产品定位与核心原则

### 1.1 产品定位

NovelScript 是面向 **个人创作者** 的 AI 创作项目平台。

> [!IMPORTANT]
> 核心价值不是"单次生成一段文本"，而是把原始文本逐步沉淀为 **可复用、可下载、可迭代的创作资产**。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **项目优先于工具** | 用户看到的是"项目"和"资产"，不是零散页面 |
| **链路优先于功能点** | 功能必须可串联，不做彼此孤立的按钮集合 |
| **中间结果可见** | 分析、大纲、剧本、分镜都应能浏览、下载、再利用 |
| **结构化优先** | 内部存储优先结构化 JSON，再渲染成人类可读文本 |
| **隐藏多租户复杂度** | 用户只感知"项目"，不感知 Workspace / Organization 层级 |

### 1.3 目标用户

| 画像 | 描述 | 核心诉求 |
|------|------|---------|
| 网文作者 | 有成品小说，希望快速改编为短剧剧本 | 低门槛、快速出稿 |
| 独立编剧 | 有剧本创作经验，需要 AI 辅助提效 | 可编辑中间产物、精细控制 |
| 短视频创作者 | 需要分镜指导来拍摄/制作内容 | 一键出分镜、下载即用 |

### 1.4 不做的事

- ❌ 工作室多人协同服务
- ❌ 团队协作 / 审批流 / 多席位
- ❌ Enterprise SSO / 私有部署
- ❌ 前台暴露 Workspace / Organization 概念

## 2. 核心流程

### 2.1 流程概览

```mermaid
graph LR
  A[上传/粘贴小说] --> B[AI 分析]
  B --> C[分集大纲]
  C --> D[剧本生成]
  D --> E[分镜提示词]
  
  A -->|一键模式| E
  D -->|手动触发| E
  
  B -.->|重新生成| C
  C -.->|重新生成| D
  D -.->|重新生成| E
  
  style A fill:#8b5cf6,color:#fff
  style E fill:#f59e0b,color:#000
```

### 2.2 流程 A：小说 → 剧本

1. 用户创建项目，选择输入类型为"小说原文"
2. 粘贴文本或上传文件（txt / md / docx），填写题材、集数、单集时长、风格
3. 系统执行：预处理 → 小说分析 → 分集大纲 → 分集剧本
4. 分析、大纲、每一集剧本分别存为独立工件，挂到当前项目下
5. 用户可浏览每个中间结果，可下载当前集剧本，也可导出全部

**验收标准：**
- [ ] 必须显示阶段化进度，而不是只显示长文本流式输出
- [ ] 剧本完成后，页面必须出现主按钮：**继续生成分镜**
- [ ] 每一集剧本都必须支持查看、复制、下载、再生成
- [ ] 分析与大纲也必须可浏览和下载

### 2.3 流程 B：剧本 → 分镜

1. 用户从项目中选择某个剧本版本（或直接导入剧本文本）
2. 可按"全剧本 / 指定集 / 指定场景"选择分镜生成范围
3. 设置视觉风格、色调、镜头密度、安全模式等参数
4. 系统先解析场景与角色，再生成分镜文本和结构化镜头数据
5. 用户可下载文本版分镜，也可导出 JSON / CSV / DOCX 等结构化结果

**验收标准：**
- [ ] 分镜生成默认绑定当前选中的剧本版本，不再依赖 URL 传输大段文本
- [ ] 角色、场景识别结果必须可见，便于用户判断抽取质量
- [ ] 失败时可在同一工作台重试，不要求跳到另一个页面

### 2.4 流程 C：小说 → 分镜（一键模式）

用户可在创建任务时直接选择"目标产物 = 分镜"，系统内部流程仍为"小说 → 剧本 → 分镜"。

1. 用户输入小说并选择目标为"分镜"
2. 系统后台自动编排：预处理 → 分析 → 大纲 → 剧本 → 分镜
3. 前端必须把中间阶段显式展示出来，不可把剧本阶段完全隐藏
4. 剧本生成成功后自动触发分镜；若剧本失败，分镜不启动
5. 生成完成后，用户可分别浏览和下载剧本、分镜

**验收标准：**
- [ ] 一键模式不能绕过剧本阶段的数据沉淀
- [ ] 分镜必须在元数据中记录依赖的剧本版本 ID
- [ ] 用户可从一键模式中途停下，在剧本阶段手动调整后再继续生成分镜

### 2.5 流程 D：项目内继续创作

任一阶段的工件都可以作为下游生成的起点：

- 从分析重新生成大纲
- 从某版大纲重新生成剧本
- 从某版剧本重新生成分镜
- 从某个分镜版本继续派生视频任务（未来阶段）

**设计原则：**
- 任一阶段都可形成可追溯、可下载、可复用的资产版本
- 项目是容器，任务是过程，工件是结果，版本是历史，关系是链路
- 用户需看到"我在什么阶段、下一步是什么、当前版本依赖什么"

## 3. 现状分析与不足之处

### 3.1 已具备能力

| 模块 | 状态 | 说明 |
|------|------|------|
| 项目管理 | ✅  | 项目 CRUD、项目工作台骨架 |

| 工件版本管理 | ✅  | parentArtifactId / versionGroupId |
| 导出功能 | ✅  | Markdown / JSON / Text |

### 3.2 不足清单

#### P0 — 必须修复

| # | 问题 | 目标 | 完成标准 |
|---|------|------|---------|
| P0-1 | **产品定位收口** — 前台还像两个独立工具，文档带 Team/Enterprise/Workspace 叙事噪音 | 收敛为个人创作者 | 首页、导航、套餐、文档不再以 Team/Enterprise 为主叙事 |

| P0-3 | **剧本→分镜联动** — 分镜仍像"把剧本文本贴进去再跑一次" | 分镜成为剧本的下游资产 | 剧本完成后出现"继续生成分镜"，默认绑定当前剧本版本 |
| P0-4 | **小说→分镜一键链路** | 满足核心流程要求 | 前台可直接选目标=分镜，后台仍按小说→剧本→分镜编排 |
| P0-5 | **资产浏览与下载** | 保留并强化可见性 | 分析、大纲、剧本、分镜都可浏览、下载、筛选 |
| P0-6 | **隐藏多租户复杂度** | 个人用户不被 workspace/org 概念打扰 | 用户只感知"项目"，不感知工作区层级 |
| P0-7 | **支付系统迁移** — Stripe → PayPal | 用 PayPal 收款 | PayPal 为唯一支付渠道，Stripe 代码完全移除 |
| P0-8 | **套餐体系调整** — Studio 计划含团队功能与定位矛盾 | 面向个人的套餐 | Free / Creator / Pro，无团队功能 |

#### P1 — 应该修复

| # | 问题 | 目标 | 完成标准 |
|---|------|------|---------|

| P1-2 | **集级/场景级选择生成** | 提高可控性与成本效率 | 用户可只为指定集或场景生成分镜 |

| P1-6 | **文件上传支持** | 除粘贴外支持 txt/md/docx 上传 | 上传后自动提取文本内容 |
| P1-7 | **阶段化进度展示** | 不再只显示长文本流式输出 | 前端展示当前执行到哪个阶段（分析/大纲/剧本/分镜） |

## 4. 功能模块 PRD

### 4.1 模块一：项目中心

**职责：** 项目的创建、列表、状态总览。

**功能：**
- 创建项目：标题、题材、简介、目标产物（剧本/分镜）
- 项目列表：按最后更新时间排序
- 项目卡片状态展示：
  - 最近更新时间
  - 是否有运行中任务
  - 已有工件类型（分析/大纲/剧本/分镜）
  - 最近任务结果（成功/失败）
- 项目操作：进入工作台、归档、删除

**不做（P0）：** 项目复制、按状态筛选（P1）

---

### 4.2 模块二：原文输入与素材管理

**职责：** 原文的输入、保存、管理。

**功能（P0）：**
- 粘贴文本输入
- 保存原始文本快照，不依赖浏览器临时会话
- 原文与项目绑定，项目内可随时回看

**功能（P1）：**
- 上传文件（txt / md / docx），自动提取文本
- 原文版本管理
- 原文标签：题材、受众、改编目标

---

### 4.3 模块三：剧本引擎

**职责：** 从原文到剧本的完整生成链路。

**功能：**
- 配置项：题材、集数、单集时长、风格
- 输出工件：小说分析、分集大纲、分集剧本（每集独立工件）
- 每次生成保存配置快照，保证版本可追溯
- 剧本完成后，页面显示主按钮「继续生成分镜」
- 按集查看、按集下载、全部导出

**功能（P2）：**
- 局部再生成：只重跑某集剧本
- 基于新大纲重跑全部剧本

---

### 4.4 模块四：分镜引擎

**职责：** 从剧本到分镜的生成链路。

**功能（P0）：**
- 输入优先级：`scriptArtifactIds` > 手动导入剧本文本
- 输出：人类可读的分镜文本
- 分镜默认绑定选中的剧本版本
- 分镜工件 metadata 记录来源剧本 ID
- 保留安全模式逻辑
- 失败时可在同一工作台重试

**功能（P1 — 分镜结构化 JSON）：**
- 同时输出结构化镜头 JSON
- 结构化字段至少包含：

```typescript
interface StoryboardShot {
  sceneId: string;
  shotId: string;
  shotType: string;      // 特写/中景/全景/...
  camera: string;        // 镜头运动
  composition: string;   // 构图
  motion: string;        // 画面运动
  subject: string;       // 主体
  environment: string;   // 环境
  lighting: string;      // 光影
  audioHint: string;     // 音效提示
  videoPrompt: string;   // 视频生成 prompt
}
```

**功能（P1 — 选择生成范围）：**
- 全剧本 / 指定集 / 指定场景

---

### 4.5 模块五：资产浏览与下载中心

**职责：** 项目内所有工件的统一浏览、筛选、下载。

**功能：**
- 统一展示项目下所有工件：原文、分析、大纲、剧本、分镜
- 展示依赖关系：本分镜来自哪一版剧本；本剧本来自哪一版大纲
- 版本选择、时间排序、按类型筛选
- 下载格式（P0）：TXT / Markdown / JSON
- 下载格式（P1）：DOCX / CSV
- 历史工件无 relation 时降级展示"无依赖数据（历史产物）"

---

### 4.6 模块六：任务中心

**职责：** 长任务的状态管理与失败处理。

**功能（P0）：**
- 任务状态：排队中、运行中、成功、失败、已取消
- 任务列表展示：触发时间、任务类型、当前状态
- 阶段化进度：展示当前执行到哪个阶段

**功能（P1）：**
- 失败时展示失败阶段、错误原因、重试入口
- 任务日志记录：触发用户、触发时间、输入快照、输出摘要、消耗用量
- Pipeline 模式下两阶段任务进度独立展示

## 6. 数据模型关键补充

### 6.1 工件关系模型（ArtifactRelation）

```typescript
interface ArtifactRelation {
  id: string;
  projectId: string;
  upstreamArtifactId: string;
  downstreamArtifactId: string;
  relationType: 'derived_from';
  metadata?: Record<string, unknown>;
  createdAt: string;
}
```

**写入时机：**
- 剧本生成完成 → `analysis → outline`、`outline → script`
- 分镜生成完成 → `script → storyboard`
- 从某版工件重新生成时 → 记录新的上下游关系

### 6.2 隐藏多租户复杂度

后端保留 `Organization / Workspace` 模型（为未来扩展预留），但前端不暴露：

- 注册时自动创建 org + workspace，用户无感知
- 导航和 UI 中不出现"工作区""组织"字样
- 项目直接展示在个人空间下

## 9. 术语表

| 术语 | 说明 |
|------|------|
| 项目 (Project) | 一次创作的容器，承载原文到最终产物的全部资产 |
| 工件 (Artifact) | 生成过程中产生的资产：分析、大纲、剧本、分镜 |
| Pipeline | 一键链路，串行执行多个 Job |
| ArtifactRelation | 上游→下游的生成依赖记录 |
| Credits | 平台内积分，用于抵扣生成消耗 |
| Job | 一次生成任务实例 |
