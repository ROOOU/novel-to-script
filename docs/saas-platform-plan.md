# NovelScript SaaS 化与商业化方案

## 当前项目判断

- 当前代码已经具备两条可商业化的核心生产链路：`小说 -> 剧本`、`剧本 -> 分镜提示词`。
- Demo 阶段的主要问题不在前端页面，而在服务端仍以单次请求为中心，缺少用户、项目、任务、计量和存储这些 SaaS 基础边界。
- 本轮代码改造先把生成编排从 API route 中抽离到 `src/server/*/application`，为后续接入账号、工作区、任务队列、账单做准备。

## 商业化定位

- 产品定位：短剧 IP 改编生产工作台，而不是泛 AI 写作工具。
- 核心价值：把原始文本沉淀为结构化生产资产，包括角色分析、分集大纲、剧本、分镜提示词。
- 目标用户：
  - 个人创作者、网文作者
  - 短剧编剧工作室、MCN 内容团队
  - 影视制作公司、IP 运营方
  - 编剧培训机构、内容教育团队

## 定价建议

- Free：限制字数、次数、导出、模板，保留体验能力。
- Pro：99-199 元/月，面向个人创作者，开放完整生成、更多配额、无水印导出。
- Team：399-999 元/月起，叠加席位和用量计费，增加模板库、协作、版本管理、项目管理。
- Enterprise：年付定制，提供专属模型配置、权限体系、审计日志、私有化部署或专属云。

## 可变现能力

- 长文本和高频生成额度
- 行业模板和题材模板
- 项目资产库和角色/世界观复用
- 团队协作、审批流、版本回溯
- 标准化导出和品牌化导出
- API、批处理、自动化生产
- 敏感内容风控、安全模式、审计追踪
- 企业级部署、SSO、专属 SLA

## 推荐模块边界

- `src/features/*`
  - 负责页面、组件、客户端状态、前端交互协议
- `src/server/*/application`
  - 负责生成编排、任务生命周期、领域用例
- `src/server/shared`
  - 负责 SSE、鉴权、租户解析、配额检查、计量、审计
- 后续新增核心实体
  - `User`
  - `Organization`
  - `Workspace`
  - `Project`
  - `SourceDocument`
  - `GenerationJob`
  - `GenerationArtifact`
  - `UsageEvent`
  - `Subscription`

## 分阶段开发路线

- Phase 1：单租户可运营版
  - 登录
  - 项目保存
  - 历史记录
  - 生成配额
  - 支付订阅
- Phase 2：标准 SaaS 版
  - 工作区
  - 团队协作
  - 任务队列
  - 对象存储
  - Usage/Billing
- Phase 3：企业版
  - SSO
  - 权限分级
  - 审计日志
  - 私有模型与私有部署

## 近期优先事项

1. 先引入持久化项目与任务表，不再依赖浏览器会话传输全文内容。
2. 把内存限流升级为租户感知的配额控制，接入 Redis 或数据库。
3. 为每次生成建立 `job` 和 `artifact` 记录，为计量、账单、重试和审计打基础。
4. 把 LLM 配置从全局环境变量扩展到 workspace 或 plan 级别的配置策略。
